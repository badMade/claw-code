## 2024-04-24 - ExecutionRegistry Lookup Optimization
- **Pattern:** Using `@functools.cached_property` on immutable `@dataclass(frozen=True)` to cache dictionary lookups.
- **Learning:** When a class does linear `O(N)` string comparisons in a loop, converting it to a lazy initialized dictionary using `@functools.cached_property` improves lookup times to `O(1)`. In the `ExecutionRegistry` class, changing `command()` and `tool()` methods to use `_command_lookup` and `_tool_lookup` cached dicts reduced lookup time from ~0.61s down to ~0.008s for 10000 iterations.
- **Context:** Because Python 3.12+ supports `cached_property` on `frozen=True` dataclasses natively, this optimization was achievable without complex state tracking. Care must be taken to only register the first element matched when duplicates are expected in the backing tuple.
